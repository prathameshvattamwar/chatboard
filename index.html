Public Chatboard Project Plan

A comprehensive overview of the development plan for a public chat platform.

Platform Overview

This platform is a public chat application where users can post text, photos, and videos, with features like commenting, liking, reacting to posts, and downloading posts if enabled by the user.

Tech Stack

  • Frontend: HTML, CSS (Bootstrap), JavaScript (jQuery)
  • Backend: Python (Flask Framework)
  • Database: SQLite for data persistence
  • Styling: Poppins font for UI
  • User Authentication: Login, registration, and authorization via Flask sessions

Database Structure

We'll need multiple tables to handle users, posts, comments, reactions, etc.

Users Table

  • id: Primary Key
  • username: Unique
  • email: Unique
  • password: Hashed
  • profile_picture: URL to the user's profile picture
  • download_enabled: Boolean (to control post download feature)
  • bio: Text (short bio)
  • created_at: Timestamp (user registration date)

Posts Table

  • id: Primary Key
  • user_id: Foreign Key (references Users table)
  • content: Text (for post text)
  • media_url: URL for photo/video upload
  • created_at: Timestamp (date of post creation)
  • downloads_enabled: Boolean (whether the post can be downloaded)
  • likes_count: Integer (total likes for the post)

User Authentication & Authorization

User registration and login are handled via Flask sessions. Only logged-in users can interact with the platform, while non-logged-in users can view posts but not interact.

User Registration

Users must provide a unique username, email, and password. Passwords should be securely hashed using tools like bcrypt or werkzeug.security.

Login

Users can log in with either their email or username. Flask sessions will manage user login states.

Authorization

Only logged-in users can post, comment, or like. Non-logged-in users can view the chat feed but cannot interact.

User Dashboard Features

The user dashboard allows users to manage their profiles, create and view posts, and interact with comments and reactions. Users can also manage their post download settings.

Profile Management

Users can upload a profile picture, update bio information, and enable/disable the option to allow downloads for their posts.

Post Management

Users can create new posts (text, images, or videos), view all past posts with likes, comments, etc., and view analytics on each post.

Comment & Reaction Management

Users can view and reply to comments and see who liked or reacted to their posts.

Followers & Following Management

Users can view and manage their followers and the users they are following.

Public Chat Features

The public chat feed displays all posts in reverse chronological order. Users can like, react, and comment on posts, with the option to download media if enabled by the post author.

Post Feed

The post feed is publicly visible and shows posts including text, images, and videos.

Likes and Reactions

Users can like or react with predefined reactions like like, love, and laugh. Reactions are aggregated and displayed next to the post.

Comments

Users can comment on posts, and their comments are shown in a threaded manner below each post.

Frontend Design

The design is responsive and mobile-friendly, utilizing Bootstrap for layout and Poppins font for styling. The chat layout includes a main feed, post creation form, and interactive elements like modals for comments.

Responsive Design

The design should be mobile-friendly using Bootstrap for a grid-based, responsive layout.

Chat Layout

The layout includes a main chat feed with user posts, media attachments, likes, reactions, and comments.

Post Creation

A user-friendly post creation form with a text box, file upload option, and submit button.

UI Styling

Use Poppins font throughout the platform for consistency. Bootstrap modals and alerts for interactivity (for creating posts, viewing comments, etc.).

Backend & API Routes

The backend is built with Flask, handling user management, post management, and analytics via API routes. Key routes include user registration, login, post creation, and fetching analytics data.

API Routes

POST /register: Register new users.
POST /login: User login.
POST /logout: Log out the user.
GET /profile: Fetch user profile and settings.
POST /profile: Update profile settings.
POST /create_post: Allow users to create new posts.
GET /posts: Fetch all posts for the chat feed.
POST /like_post: Like a post.
POST /comment_post: Comment on a post.
GET /post/<id>: Fetch a specific post's details.
GET /post/<id>/analytics: View analytics for a particular post (likes, views, downloads).
GET /download_post/<id>: Download post media if the author has enabled it.
POST /follow_user/<user_id>: Follow the user with the specified `user_id`.
DELETE /unfollow_user/<user_id>: Unfollow the user with the specified `user_id`.
GET /followers: Get the list of users following the logged-in user.
GET /following: Get the list of users the logged-in user is following.

User Analytics

Analytics are provided at both the user and post levels, offering insights into user activity, post engagement, and download statistics.

User-Level Analytics

View the number of posts they have created, total likes and comments received on their posts, and the number of downloads enabled (if they opted for media download).

Post-Level Analytics

View the number of views, likes, comments, and downloads on each post.

Testing & Deployment

The application will undergo unit testing, UI testing, and security testing before deployment on platforms like Heroku or AWS. Static files will be hosted on services like AWS S3 or Cloudinary.

Testing

Unit Testing: Write tests for key routes like user registration, login, post creation, and interactions.

UI Testing: Ensure the frontend components (forms, buttons, etc.) are working correctly on different devices.

Security Testing: Test for SQL injection, XSS attacks, and CSRF vulnerabilities.

Deployment

Use Heroku or AWS for deployment of the Flask app. SQLite can be used in development, but for production, consider switching to PostgreSQL or MySQL.

Static File Hosting: Use AWS S3 or Cloudinary for storing media files (photos/videos).

Version Control: Use GitHub to manage the code and track progress.
